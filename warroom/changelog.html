v3.4.32 ★ Major release★ 重大版本 2026-06-20

Memory → Understanding从记忆到理解

The shift from a memory system (collect & recall) to an understanding system (anticipate). Two structural moves: a two-tier memory (lossless archive under a compressed model, joined by a confidence bridge) and predict-the-correction — surfacing a warning before you have to give it. Five waves, each fact-checked against the live code, adversarially verified, and code-reviewed.从记忆系统（收集与召回）迈向理解系统（预判）。两招:双层记忆（无损档案层 + 压缩模型层,以置信度桥相连）与 predict-the-correction —— 在你开口纠正之前就把警告推出来。五个 Wave,每个都对照真实代码核实、对抗式验证并经 code-review。

🧠

Two-tier memory + bridge双层记忆 + 桥

A lossless verbatim archive (journal/archive/raw/) written on every session end, under a lossy compressed model. When confidence is low, recall drills down to the verbatim source instead of bluffing.每次会话结束写入无损逐字档案（journal/archive/raw/），其上是有损的压缩模型。置信度低时,召回会回钻到原文,而不是硬编一个不确定的答案。

Why: a floor nothing falls through, plus a model that knows when to fall back.为什么:一个谁也漏不下去的地板,加上一个知道何时回退的模型。

🔒

Privacy sync gate隐私同步门控

Plugged a live leak: the behavioral/awareness layer was syncing to Supabase on every write. New classification.ts (personal vs project) gates it; sync_personal=false by default.堵住一个正在发生的泄漏:行为/awareness 层此前每次写入都同步到 Supabase。新增 classification.ts(personal vs project)做门控,默认 sync_personal=false。

Why: your behavioral model must not leave the machine by default.为什么:你的行为模型默认不该出本机。

♻️

Compression that compounds会复利的压缩

Revived the dormant FSRS reinforce-on-recall loop (throttled), added an in-repo decay pass, made the archived flag actually filter, and a crystallization-candidate detector.复活沉睡的 FSRS reinforce-on-recall 回路(带节流),新增仓内 decay pass,让 archived 标志真正生效过滤,并加入结晶候选检测。

Why: what you use survives; what you don't fades. Collect → compress into rules.为什么:用得多的留存,用不到的褪色。从堆积转向压缩成规则。

🧭

Predict-the-correction预测纠正

An authoritative P0 correction now overrides a conflicting plan (verdict:'blocked'). A Blind-Spots profile is auto-derived from accumulated corrections (personal tier, sync-excluded); predictCorrection pushes it as an early prior.权威 P0 纠正现在会压倒冲突的计划(verdict:'blocked')。从累积纠正自动推导 Blind-Spots 画像(personal 层,默认不同步);predictCorrection 把它作为提前先验注入。

Why: memory recalls the past; understanding anticipates the next correction.为什么:记忆调取过去,理解预判下一次纠正。
