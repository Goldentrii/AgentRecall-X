The Loop — Plywood SOP (run 6×)
SOP run_loop(N, plan):
  ASSERT branch == "opt/autonomous-loops"  AND  no_push AND no_publish AND no_version_bump   # REDLINE

  Phase 1 — IMPLEMENT
    impl  = WORKER.implement(plan)                 # small, single-purpose change on branch
    gate  = RUN("npm run build && npm test")
    IF gate.red:
      FIX(once); gate = RUN(...)                    # retry once
      IF still red: REVERT(impl); loop.status = BLOCKED   # carry the failure to the round-table as the finding
    COMMIT(local, "loop N: " + plan)                # never push

  Phase 2 — ROUND-TABLE (parallel, diverse, must TEST not just opine)
    results = PARALLEL[
      TESTER.run(build/test/eval) -> real stdout,
      SKEPTIC.attack(impl)        -> "did it fix the thing or just rename it?",
      RESEARCHER.investigate(loop.theme) -> papers / math / linguistics grounding,
    ]

  Phase 3 — SYNTHESIZE
    synth = CHAIR(results) -> { worked, still_broken, eval_numbers, next_loop_design }

  Phase 4 — ORCHESTRATOR DECISION (me, bird's-eye)
    UPDATE_HTML(loop N)  ;  plywood_feedback(sop_id=946850b5, outcome, note)
    RETURN synth.next_loop_design          # drives loop N+1

⑩ 🎨 Intent-convergence experiment done ✓ — UNTESTABLE / refuted

Theme: creative · math — making MATH.md §d falsifiable. Commit 5e38609. Build/test 493/493.

🎯 The answer to "how does math help" — with evidence, not a slogan

The experiment models repeated same-cluster corrections as noisy samples of a latent intent and measures whether the estimate converges (Jaccard plateau / marginal-novelty→0 / consensus-SNR rise) as N grows. On the real corpus:

active clusters with N≥3: 0 (size dist {1:12, 2:5}) → UNTESTABLE. The only N≥3 clusters (4, incl. retracted) are boilerplate-tag artifacts — they dissolve to size 1 under rule-only signatures, and where measurable they REFUTE (mean consensus-SNR 0.035 ≪ 0.25 floor; no shared core).

The honest conclusion: "redundancy over time reconstructs intent" is not supported by this corpus — and can't even be tested, because the corpus is intent-sparse (users mostly correct each thing once). This is the third independent confirmation of one ceiling: L3 (0/13), L9 (0/25), L10 (untestable). The bottleneck is data density per intent, not the matching algorithm.

What would make it true: (a) genuine redundancy — multiple distinct corrections of the same intent per project (a function of usage depth), OR (b) a learned representation (embeddings) so semantically-equivalent-but-differently-worded corrections cluster without relying on token/tag overlap. Token-overlap clustering is structurally incapable of it — which is exactly why the only clusters it found were tag artifacts.

Self-evaluation

The experiment at its most valuable — 9.5/10. It refused to claim a result the data doesn't contain: it distinguished UNTESTABLE from REFUTED, used 4 diagnostic views to expose tag-inflation, documented its own ceil(k/2) Jaccard artifact, and the synthetic test proves the instrument discriminates real convergence from noise. A faked "supported" would have been worthless; this located the real reason understanding isn't emerging yet and pointed at the two concrete fixes. This is what the whole run was for.
